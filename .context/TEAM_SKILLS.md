## 4. AI/LLM подход

### Принцип: Облако ↔ Локально

Каждый проект разрабатывается с возможностью работы **как на облачных, так и на локальных LLM**.

| Этап | Рекомендация |
|------|--------------|
| MVP, прототип | Облачные LLM (быстрый старт) |
| Production | Выбор по требованиям (стоимость, приватность, latency) |
| Чувствительные данные | Локальные LLM приоритет |

### LLM-провайдеры

| Провайдер | Тип | Назначение |
|-----------|-----|------------|
| **Anthropic** (Claude) | Облако | Основной для сложных задач |
| **OpenAI** (GPT) | Облако | Альтернатива, embeddings |
| **Ollama** | Локально | Llama, Mistral, и др. |
| **vLLM, llama.cpp** | Локально | Высокопроизводительный inference |

### Embeddings

| Провайдер | Тип |
|-----------|-----|
| OpenAI text-embedding-3 | Облако |
| Ollama (nomic-embed, mxbai) | Локально |
| sentence-transformers | Локально |

### Абстракция

- **Esperanto** — унифицированный интерфейс к LLM
- **LangChain** — цепочки, интеграции
- **LangGraph** — stateful агенты

**Цель:** Код не привязан к конкретному провайдеру. Переключение облако ↔ локально — через конфиг.

## 5. Хранение данных

### Принцип: Безопасность данных клиента

Приоритет — защита информации. Выбор хранилища зависит от требований проекта.

| Тип | Варианты |
|-----|----------|
| Файлы | S3-совместимое (MinIO self-hosted), локальное хранение |
| Векторы | Qdrant (self-hosted), локальные индексы |
| БД | PostgreSQL, SurrealDB — self-hosted |

### Подход

- **MVP:** Можно облачные хранилища
- **Production с чувствительными данными:** Self-hosted, шифрование
- **Гибридный:** Публичные данные в облаке, приватные локально
