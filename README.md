# Voice Dictation

Голосовой ввод текста для Windows. Нажми клавишу — говори — текст вставляется в любое приложение.

Распознавание работает **полностью локально** через [faster-whisper](https://github.com/SYSTRAN/faster-whisper) на NVIDIA GPU. Интернет не нужен. Данные не покидают компьютер.

## Возможности

- **Push-to-talk** — удерживай горячую клавишу, говори, отпусти — текст вставится в активное поле
- **Полностью офлайн** — всё работает локально, без облака и интернета
- **Русский + английский** — автоопределение языка, смешанная речь
- **LLM-коррекция** — опциональная постобработка через Qwen2.5 для улучшения пунктуации и терминов
- **Словари терминов** — доменные словари (IT, математика, инженерия) для точного распознавания
- **Постобработка текста** — автоматическая пунктуация, капитализация, точка в конце
- **Управление моделями** — скачивание и выбор моделей Whisper через UI
- **Минималистичный виджет** — компактная полоска-индикатор на экране
- **Системный трей** — сворачивается, не мешает работе

## Два режима работы

Voice Dictation работает в двух режимах в зависимости от того, включена ли LLM-постобработка.

### Режим 1: Без LLM (по умолчанию)

Базовый режим. Whisper распознаёт речь, regex-процессоры нормализуют текст.

| Компонент | Описание |
|-----------|----------|
| **Whisper** | Распознавание речи (large-v3 по умолчанию) |
| **Фильтр галлюцинаций** | Отсеивает мусорные фразы Whisper |
| **Словарь замен** | `replacements.json` — фикс часто неверных слов |
| **Regex-обработка** | Пробелы вокруг пунктуации, заглавные буквы, точка в конце |

**VRAM:** ~3.5 GB (модель large-v3 в float16). Подходит для любой NVIDIA GPU с 4+ GB. С моделью large-v3-turbo — ~2.5 GB.

**Когда подходит:**
- Любая NVIDIA GPU с 4+ GB VRAM
- Быстрая диктовка без задержек
- Текст не требует идеальной пунктуации

### Режим 2: С LLM-коррекцией

Расширенный режим. После Whisper текст проходит через LLM (Qwen2.5-1.5B), которая исправляет пунктуацию, капитализацию и ошибки распознавания. Regex-обработка работает **после** LLM — они дополняют друг друга.

| Компонент | Описание |
|-----------|----------|
| **Whisper** | Распознавание речи (large-v3 по умолчанию) |
| **Фильтр галлюцинаций** | Отсеивает мусорные фразы Whisper |
| **LLM Qwen2.5-1.5B** | Исправляет пунктуацию, капитализацию, ошибки |
| **Словарь замен** | `replacements.json` — фикс часто неверных слов |
| **Regex-обработка** | Нормализация пробелов и финальная пунктуация |

**VRAM:** ~11 GB суммарно (Whisper large-v3 ~3.5 GB + LLM ~7.5 GB). Это **пиковое** потребление — в простое модели почти не занимают ресурсов. Для комфортной работы достаточно GPU с 12 GB VRAM (RTX 3060 12GB, RTX 4070 и выше).

**Когда подходит:**
- GPU с 12+ GB VRAM
- Нужна качественная пунктуация и капитализация
- Технические тексты с терминологией

> **Примечание:** LLM по умолчанию работает в float32. На некоторых конфигурациях float16 генерирует некорректный вывод при совместной работе с Whisper. Приложение автоматически проверяет работоспособность и при проблемах откатывается на float32. LLM опциональна — без неё приложение работает прекрасно.

## Требования

- **Windows 10/11**
- **NVIDIA GPU** с поддержкой CUDA
  - Без LLM: 4+ GB VRAM — подходит большинство современных GPU
  - С LLM: 12+ GB VRAM (RTX 3060 12GB, RTX 4070 и выше)
- **Microsoft Visual C++ Redistributable** (обычно уже установлен)

### Потребление VRAM

| Конфигурация | VRAM | Пример GPU |
|-------------|------|-----------|
| Whisper large-v3 (float16) | ~3.5 GB | RTX 3060, RTX 4060 |
| Whisper large-v3-turbo (float16) | ~2.5 GB | GTX 1660, RTX 3060 |
| + LLM Qwen2.5-1.5B (float32) | ~11 GB | RTX 3080, RTX 4070 |
| Whisper medium (float16) | ~1.5 GB | GTX 1060 |

```
cd python311

```

```
install_local.bat
```

## Использование

1. Запусти Voice Dictation
2. На экране появится компактный виджет-полоска (зелёный = готов)
3. **Удерживай F9** — идёт запись (виджет красный)
4. **Отпусти F9** — обработка (жёлтый) — текст вставляется в активное поле
5. Готово. Виджет снова зелёный — можно диктовать дальше

### Управление

| Действие | Как |
|----------|-----|
| Перетащить виджет | ЛКМ + перетащить |
| Контекстное меню | ПКМ на виджете |
| Настройки | ПКМ — «Настройки...» |
| Управление моделями | Настройки — вкладка «Модели» |
| Свернуть в трей | ПКМ — «Свернуть в трей» |
| Развернуть из трея | Двойной клик по иконке в трее |

## Настройки

Все настройки доступны через UI: **ПКМ на виджете — «Настройки...»**. Конфигурация хранится в `config.json` (создаётся автоматически).

### Вкладка «Основные»

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| **Горячая клавиша записи** | Клавиша для push-to-talk. Нажми на поле, затем нажми нужную клавишу или комбинацию. Поддерживаются: F-клавиши, буквы, комбинации (Ctrl+Space, Shift+Tab, Ctrl+Tab), а также отдельные модификаторы (Left Ctrl, Right Shift) | `F9` |
| **Язык распознавания** | Язык речи. «Авто» — определяет автоматически. Фиксированный язык ускоряет распознавание | `Авто` |
| **Запускать при старте Windows** | Автозапуск при входе в систему | Выкл |
| **Запуск в свёрнутом виде** | Запуск сразу в трей, без виджета на экране | Выкл |
| **Усиление микрофона** | Коэффициент усиления входного сигнала. 1.0 = без усиления. Увеличь, если микрофон тихий | `1.0` |
| **Звуковые эффекты при записи** | Звуковой сигнал при начале и окончании записи | Вкл |
| **Приглушать звук при записи** | Уменьшение громкости системных звуков во время записи, чтобы они не попадали в микрофон | Вкл |

### Вкладка «Распознавание»

#### Группа «Модель»

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| **Модель** | Модель Whisper для распознавания. Требует перезапуск | `large-v3` |
| **Устройство** | CUDA (GPU) или CPU. GPU значительно быстрее. Требует перезапуск | `CUDA (GPU)` |
| **Точность** | Числовая точность модели: float16 (быстро, меньше VRAM), int8_float16, int8 (минимум VRAM). Требует перезапуск | `float16` |

#### Группа «Параметры качества»

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| **Ширина поиска (beam)** | Ширина beam search. Больше = точнее, но медленнее | `5` |
| **Использовать предыдущий сегмент как контекст** | Передавать предыдущий сегмент как контекст для следующего. Для push-to-talk лучше выключено | Выкл |
| **Порог сжатия** | Фильтр повторяющегося текста. Сегменты с высоким коэффициентом сжатия отбрасываются | `2.4` |
| **Порог вероятности** | Минимальная средняя log-вероятность сегмента. Сегменты с низкой уверенностью отбрасываются | `-1.0` |
| **Порог тишины** | Вероятность «нет речи». Сегменты выше порога считаются тишиной и пропускаются | `0.6` |
| **Штраф повтора** | Штраф за повторение токенов. >1.0 снижает повторения | `1.2` |
| **Размер N-грамм** | Запрет повтора N-грамм подряд. 0 = без ограничений | `3` |
| **Фильтр галлюцинаций** | Длительность тишины (сек), после которой текст считается галлюцинацией и отбрасывается | `2.0` |

#### Группа «VAD (детектор речи)»

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| **Чувствительность** | Порог детекции речи (0.0–1.0). Меньше = чувствительнее | `0.50` |
| **Мин. длина речи** | Минимальная длительность речевого сегмента (мс) | `250 мс` |
| **Мин. длина паузы** | Минимальная длительность паузы для разделения сегментов (мс) | `500 мс` |

### Вкладка «Постобработка»

#### Группа «LLM-коррекция»

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| **Использовать LLM для коррекции текста** | Включает LLM-постобработку через Qwen2.5-1.5B. При первом включении предложит скачать модель (~3 GB). Требует ~7.5 GB дополнительной VRAM | Выкл |
| **Скачать модель** | Кнопка для скачивания и конвертации LLM-модели. Появляется если модель не установлена | — |

LLM исправляет:
- Пунктуацию (запятые, точки, тире)
- Капитализацию (начало предложений, имена собственные)
- Очевидные ошибки распознавания

#### Группа «Regex-обработка»

Regex-обработка работает **всегда** (если включена), в том числе вместе с LLM — они дополняют друг друга.

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| **Нормализация пробелов вокруг знаков препинания** | Убирает лишние пробелы перед `.,:;!?` и добавляет после них | Вкл |
| **Заглавная буква в начале и после .!?** | Первая буква текста и буква после точки/восклицательного/вопросительного — заглавная | Вкл |
| **Добавлять точку в конце** | Если текст не заканчивается на `.!?`, добавляет точку | Вкл |

### Вкладка «Словари»

Доменные словари улучшают распознавание специфических терминов через механизм hotwords.

| Словарь | Содержимое |
|---------|-----------|
| **IT** | Термины программирования, фреймворки, технологии |
| **Math** | Математические термины и обозначения |
| **Engineering** | Инженерная терминология |

Можно включать несколько словарей одновременно. Термины передаются модели Whisper как подсказки для распознавания.

Дополнительно: файл `dictionary.txt` в корне — базовый словарь, применяется всегда.

### Вкладка «Модели»

Таблица всех доступных моделей Whisper с возможностью скачивания и переключения.

| Модель | Размер | Качество | VRAM |
|--------|--------|----------|------|
| tiny | ~0.1 GB | Базовое | ~1 GB |
| base | ~0.2 GB | Низкое | ~1 GB |
| small | ~0.5 GB | Среднее | ~2 GB |
| medium | ~1.5 GB | Высокое | ~5 GB |
| large-v3-turbo | ~1.5 GB | Отличное | ~2.5 GB |
| **large-v3** | ~3 GB | **Лучшее** | ~3.5 GB |

**large-v3** — модель по умолчанию, лучшее качество. **large-v3-turbo** — быстрее при сопоставимом качестве, подойдёт для GPU с ограниченной VRAM.

## Рекомендуемые настройки

### Для GPU с 4–8 GB VRAM (GTX 1660, RTX 3060 8GB, RTX 4060)

- Модель: `large-v3` (float16) — ~3.5 GB, или `large-v3-turbo` — ~2.5 GB
- LLM-коррекция: **выключена**
- Все regex-процессоры: включены
- Словари: по необходимости
- Распознавание работает отлично и без LLM

### Для GPU с 12+ GB VRAM (RTX 3060 12GB, RTX 4070, RTX 4080, RTX 5090)

- Модель: `large-v3` (float16)
- LLM-коррекция: **включена**
- Все regex-процессоры: включены
- Словари: все нужные домены

## Словарь замен

Файл `replacements.json` в корне — словарь принудительных замен. Применяется после распознавания (и после LLM, если включена). Формат:

```json
{
  "неправильное слово": "правильное слово",
  "питон": "Python",
  "гитхаб": "GitHub"
}
```

Замены регистронезависимые, работают по границам слов.

## Решение проблем

**Виджет не появляется**
Проверь `logs/dictation.log`. Возможно, не хватает VRAM для загрузки модели.

**GPU не используется**
Убедись что установлены драйверы NVIDIA с поддержкой CUDA. Или смени устройство на CPU в настройках (будет медленнее).

**Плохое распознавание**
- Добавь термины в `dictionary.txt` или включи доменные словари
- Попробуй увеличить beam size (5 → 7)
- Включи LLM-коррекцию (если хватает VRAM)

**Нет звука / не слышит микрофон**
- Проверь что микрофон выбран по умолчанию в настройках Windows
- Попробуй увеличить усиление микрофона в настройках (> 1.0)

**Не хватает VRAM**
- Используй модель поменьше (large-v3-turbo или medium вместо large-v3)
- Выключи LLM-коррекцию
- Смени точность на int8 или int8_float16

**LLM-коррекция не работает**
- Проверь что модель скачана (Настройки — Постобработка — «Скачать модель»)
- Проверь VRAM — нужно ~10 GB суммарно для Whisper + LLM
- Посмотри лог — приложение автоматически откатывается на float32 если float16 не работает

**Текст не вставляется в приложение**
- Voice Dictation использует unicode SendInput для вставки — работает в большинстве приложений
- Для Electron-приложений (VS Code, Slack) вставка через SendInput поддерживается с v1.0.3
